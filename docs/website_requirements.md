# ホームページ要件定義書

- 対象サイト: わたたたるお合同会社 公式ホームページ
- 作成日: 2026-04-20
- リポジトリ: `watatataruo/site`
- 公開URL: `https://watatataruo.github.io/site/`
  - 現行の GitHub Pages 構成と `vite.config.js` の `base: '/site/'` に基づく公開URL

## 1. このホームページの目的

本サイトの目的は、わたたたるお合同会社の事業内容、考え方、実績、会社情報を簡潔に伝え、相談や依頼につながる導線を提供することです。

主目的は以下です。

- 会社の存在と提供価値を分かりやすく伝える
- AI活用支援、情報整理、発信設計などの事業内容を紹介する
- 代表者の考え方と会社の姿勢を伝える
- note / X の発信導線を整える
- お問い合わせフォームから相談を受け付ける

## 2. 想定ユーザー

想定ユーザーは以下です。

- AIを業務に取り入れたい個人事業主、小規模事業者、法人担当者
- 情報整理、資料作成、発信設計に課題を感じている人
- 代表者の考え方や発信内容を見て相談を検討する人
- 会社情報や基本的な信頼情報を確認したい人

サイトは大企業向けの大規模コーポレートサイトではなく、小規模で誠実な事業者サイトとして設計する。

## 3. サイト構成

現状のサイトは React + Vite で実装されたシングルページアプリケーションであり、URL ルーティングではなく React state による画面切り替えで構成される。

主要な構成は以下。

- `index.html`
  - 基本メタ情報、OGP、Twitterカード、フォント preconnect を管理
- `src/main.jsx`
  - React エントリーポイント
- `src/App.jsx`
  - サイト本体
  - ページコンポーネント、共通レイアウト、SEO更新、問い合わせフォーム実装を内包
  - 大半のスタイルは `CSS` 文字列として同ファイル内に定義されている
- `src/index.css`
  - 最低限のグローバルスタイル
- `vite.config.js`
  - GitHub Pages 用 `base: '/site/'` を設定
- `public/`
  - favicon など静的アセット

補足事項。

- 画面遷移は React Router ではなく `page` state で切り替えている
- そのため、公開上のURLは基本的に 1 つで、ページごとの個別URLは持たない
- `gas-code.gs` は問い合わせ受信を担う別管理の Google Apps Script 側コードであり、このリポジトリには含まれない場合がある

## 4. 各ページの役割

### 4.1 トップ

- サイト全体の要約ページ
- 会社の価値提案、事業概要、価値観、発信・実績導線、お問い合わせ導線をまとめる
- 最初に「何をしている会社か」を短時間で理解できることが重要

### 4.2 事業内容

- 各サービスの概要を詳細に説明するページ
- 向いている課題と相談イメージを整理して提示する
- 「依頼前の整理」に役立つ情報ページとして機能する

### 4.3 代表メッセージ

- 代表者の考え方、会社を立ち上げた背景、仕事観を伝える
- サービス説明だけでは伝わらない信頼感を補強する

### 4.4 発信・実績

- note 記事、実績、SNS 発信をまとめるページ
- 実績量よりも、考え方や取り組みの継続性を見せる役割を持つ

### 4.5 会社概要

- 会社名、英語表記、法人形態、所在地、設立、代表者、事業内容、SNS などの基本情報を整理して掲載する
- 正式な会社情報ページとして扱う

### 4.6 プライバシーポリシー

- 問い合わせ時に取得する個人情報の取り扱いを明記する
- お問い合わせフォームと対になる説明責任ページ

### 4.7 お問い合わせ

- ご依頼、ご相談、ご質問の受付ページ
- フォーム送信と SNS 代替導線の両方を持つ
- 実装・運用上の重要ページであり、変更は慎重に扱う

## 5. デザイン方針

本サイトのデザイン方針は「落ち着き」「読みやすさ」「誠実さ」「小規模事業者としての信頼感」を重視すること。

具体方針は以下。

- 白と淡い背景色をベースに、ネイビーをアクセントに使う
- Noto Sans JP / Noto Serif JP を中心に、和文の可読性と品位を確保する
- 本文は左揃えを基本とし、CTA など必要な箇所のみ中央配置を使う
- 余白は広すぎず詰めすぎず、文章量に対して読みやすい密度に整える
- カード、表、導線ボタンは整列を優先し、雑然と見せない
- スマホ表示でも不自然な改行や詰まりが起きないよう、レイアウトはレスポンシブ前提で維持する

## 6. 文体方針

文体は以下を基準とする。

- 丁寧だが過剰に営業的ではない
- 難しい言葉を避け、平易で具体的な日本語を使う
- 断定しすぎず、相談しやすさを感じるトーンにする
- 「誠実」「整理」「伝わる」を軸に据える
- 大げさな実績誇張や煽り表現は避ける

## 7. 会社概要の扱い

会社概要は正式情報として扱い、内容変更時は確認を要する。

現状の掲載方針は以下。

- 会社名: `わたたたるお合同会社`
- 英語表記: `Watatataruo LLC`
- 法人形態: `合同会社（LLC）`
- 所在地: `インターネット`
- 設立: `2024年`
- 連絡先の直接公開は行わず、お問い合わせフォームへ誘導する

注意事項。

- 電話番号、メールアドレス、詳細住所を勝手に追加しない
- 会社情報の追加公開は、事業者判断が入る前提で扱う

## 8. 代表者名の扱い

代表者名は `わたたたるお` として扱う。

現状の表示方針は以下。

- 代表メッセージページでは `代表　わたたたるお`
- 役職表記は `わたたたるお合同会社 代表社員`

注意事項。

- 別表記、別名義、ローマ字名を勝手に追加しない
- 敬称、肩書き、プロフィール詳細は現行トーンを崩さない範囲で扱う

## 9. SNSリンクの方針

現状の公式導線は以下の 2 つ。

- note: `https://note.com/watatata00`
- X: `https://x.com/watatata00`

方針は以下。

- SNS リンクは公式導線として扱う
- 外部リンクは新しいタブで開き、`rel="noreferrer"` を付与する
- フッター、発信・実績ページ、会社概要、お問い合わせ補助導線で再利用する
- 未確認の SNS や別サービスを勝手に追加しない

## 10. 問い合わせフォームの構成

お問い合わせフォームは以下の構成を持つ。

- お名前
  - 必須
- メールアドレス
  - 必須
- お問い合わせ種別
  - 任意
  - 選択肢:
    - サービス・料金について
    - 案件のご相談
    - 取材・コラボレーション
    - その他
- お問い合わせ内容
  - 必須

画面上の振る舞い。

- 必須未入力時はエラー表示を行う
- メール形式チェックを行う
- 送信中はローディング状態を表示する
- 成功時は送信完了表示を出す
- 補助導線として X / note リンクも併記する

## 11. 問い合わせフォームの GAS 連携

問い合わせフォームは Google Apps Script と連携している。

実装上の前提は以下。

- フロントエンド側は `src/App.jsx` 内の `GAS_URL` に対して POST 送信する
- 送信形式は `application/x-www-form-urlencoded`
- 送信データは `name`, `email`, `kind`, `body`
- `redirect: "follow"` で GAS のリダイレクト先まで追う
- 応答は JSON を期待し、`result === "success"` を成功条件として扱う
- 受信側ロジックは `gas-code.gs` 側で管理される

## 12. 問い合わせフォーム関連は慎重に扱うこと

問い合わせフォームは現在正常動作している前提であり、変更時のリスクが高い。以下は高リスク領域として扱う。

- `App.jsx` 内の `Contact` コンポーネント本体
- `GAS_URL`
- `handleSubmit` を含む送信処理
- `gas-code.gs` 側の問い合わせ受信処理
- フォーム入力欄
- 送信ボタン
- 送信完了表示
- エラー表示
- フィールド名 `name`, `email`, `kind`, `body`

原則。

- 問い合わせフォーム関連は、依頼内容に明示されていない限り変更しない
- フォーム関連を変更する場合は、フロントと GAS の両方をセットで確認する
- 見た目調整の依頼でも、問い合わせフォームまわりは原則対象外として扱う

## 13. SEO の基本方針

現状の SEO 方針は「基本を外さない最小限のSEO」である。

実装済みの基本要素。

- `html lang="ja"`
- title
- meta description
- robots
- theme-color
- OGP
- Twitter card
- Organization の JSON-LD

実装上の注意。

- 現状は React state による画面切り替えで、ページごとの個別URLを持たない
- `pageMeta` により画面切り替え時の title / description は更新しているが、URL単位のSEOには限界がある
- 本格的に検索流入を強化する場合は、将来的にルーティング導入や静的ページ分割を検討する

SEO文言の方針。

- 誇張よりも説明の明確さを優先する
- 会社名、事業内容、提供価値が自然に分かる文言にする
- note / X など外部発信の存在は補助情報として扱う

## 14. GitHub Pages / リポジトリ運用方針

現行運用は GitHub Pages を前提とする。

- リポジトリ: `https://github.com/watatataruo/site`
- 公開URL: `https://watatataruo.github.io/site/`
- Vite の `base` は `/site/`

運用上の前提。

- GitHub Pages の project site として動かす前提で、相対パスや base 設定を崩さない
- 公開URLが変わる場合は `vite.config.js` と本書を同時に更新する
- カスタムドメインを導入した場合も、base 設定との整合性確認が必要

## 15. 今後の修正時に気をつけること

今後の修正では以下を守る。

- 問い合わせフォーム関連は依頼がない限り触らない
- 大半の見た目は `src/App.jsx` 内の `CSS` 文字列で管理されていることを前提に編集する
- SPA 構成のため、ページ追加やSEO改善は `App.jsx` の state 切り替えと `pageMeta` の更新をセットで考える
- 改行は必要最小限にし、不自然な `<br />` を増やしすぎない
- スマホ表示で不自然にならないかを常に確認する
- フッターやカードなど、整列が崩れやすい箇所は横並びと縦位置の両方を確認する
- 会社情報、代表者名、SNS URL は事実情報として扱い、推測で変更しない
- 外部リンクは安全な属性を維持する
- SEO 変更時は `index.html` と `App.jsx` の両方を確認する

## 16. 将来AIへ渡す際の指示要約

このサイトを編集する AI / 開発者には、最低限以下を共有すること。

- このサイトは GitHub Pages 上の React + Vite SPA である
- レイアウトとスタイルの中心は `src/App.jsx`
- 問い合わせフォームは GAS 連携済みの稼働領域であり、原則として不用意に変更しない
- SEO は最小限運用であり、強化には構成変更の検討が必要
- 会社情報と代表者表記は勝手に増やしたり変えたりしない
